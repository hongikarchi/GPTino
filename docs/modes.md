# Plan / Auto 모드 정의 — 2026-07-24

사용자 논의로 확정된 방향. 현재 구현과의 격차를 명시해 다음 구현 라운드의 기준으로 삼는다.

## 기본 원칙 (확정)

GPTino는 **기본적으로 자율 주행**한다. 사용자의 승인을 단계마다 받지 않는다.
안전은 승인 다이얼로그가 아니라 신뢰성 계층이 담당한다: 단일 작성자 브로커, fingerprint
동시성 게이트, 서버 검증(모델 자기신고 불인정), managed history 롤백. 이 원칙 때문에
Codex 자체 승인(approvalPolicy)은 의도적으로 `never`로 꺼져 있다 — 이것은 버그가 아니라
설계다. 승인 권한을 브로커 게이트로 옮긴 것.

## Auto 모드 (기본)

- 메시지를 받으면 계획-실행-검증을 한 턴 루프 안에서 자율 수행한다.
- 라이브 문서 쓰기는 전부 `change_submit` → 브로커 게이트 → 검증 → 커밋 경로.
- 사용자 개입 지점: 일시정지(세션/전체), Stop current, 그리고 conflict/blocked 카드의
  문제-해결 안내에 따른 후속 지시.

## Plan 모드 (목표 정의 — 미구현)

Plan 모드는 "쓰기 금지 모드"가 아니라 **더 비싼 사고 + 승인 게이트 + 자동 전환**이다:

1. **더 구체적인 계획**: plan 세션의 턴은 상향된 reasoning effort로 라우팅한다
   (라우팅 정책에 plan 보정 추가). 계획 전용 지시문을 주입해 단계·대상 리소스·검증
   기준·리스크를 명시한 계획 산출물을 만들게 한다.
2. **사용자에게 요약 제시**: 계획 완료 시 패널에 계획 요약 카드 + **승인 버튼**을 띄운다.
3. **승인 → Auto 전환 → 실행**: 승인 버튼을 누르면 세션이 auto로 전환되고, 방금 승인된
   계획을 컨텍스트로 실행 턴이 시작된다. 거부/수정 시 plan 모드에 머문다.

### 현재 구현 (2026-07-24 기준)

- plan/auto는 세션 `role`(planner/modeler)로 저장되고, plan의 효과는 단 하나:
  `change_submit`이 거부된다 (`DynamicToolDispatcher.SubmitChangeAsync`).
- 계획 전용 지시문 없음, effort 보정 없음, 계획 산출물 없음, 승인 버튼/전환 플로 없음.

### 구현 시 필요한 조각 (예상 지점)

- 라우팅: `MessageRoutingPolicy`/`AdaptiveModelRouting`에 plan 보정.
- 지시문: `assets/instructions/`에 plan 전용 지시문(리빌드 없이 실험 가능).
- 산출물: 계획을 artifact로 저장(기존 artifact 경로 재사용) + 세션에 "계획 대기" 상태.
- 패널: 계획 요약 카드 + Approve/Revise 버튼 → `PUT /sessions/{id}/mode` + 실행 트리거.

## 참고 — "터미널 codex/fable보다 성능이 낮다"의 구조적 이유

Claude 백엔드 논의와 별개로, 현 구조상 다음이 체감 성능 차이를 만든다:

1. **sandbox=read-only + MCP 전부 차단**: 에이전트가 쓸 수 있는 건 gptino_v1 typed 툴뿐.
   터미널 세션이 쓰는 파일 쓰기/임의 명령 실행 자유도가 없다 (의도된 제약).
2. **브리지 타임아웃과 UI 스레드 정체**: 무거운 GH solve가 Rhino UI 스레드를 점유하면
   브리지 op가 45초에서 포기되고, 에이전트 입장에서는 툴 호출이 "실패"로 보인다 —
   중간 에러/포기의 주요 원인. (freeze 완화가 곧 체감 성능 개선.)
3. **게이트 거부가 에러로 보임**: fingerprint 불일치/stale 거부는 신뢰성 계층의 정상
   동작이지만 모델에게는 실패 신호라 재시도 대신 포기를 고를 수 있다. 거부 메시지의
   해결 지시문 개선(이번 라운드의 resolution 필드)이 이 부분을 겨냥한다.
4. **자동 라우팅의 하향**: auto 프로필이 작은 모델/낮은 effort를 고르는 경우 터미널에서
   수동으로 큰 모델을 쓰던 체감과 차이가 난다. Quality를 deep으로 고정하면 좁혀진다.
